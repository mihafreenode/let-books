# Übersetzungsspeicher

Translation Memory ist ein System, das zuvor übersetzte Segmente speichert, damit sie wiederverwendet werden können, wenn derselbe oder ein ähnlicher Text erneut erscheint. In Dokumentation, Produkt-UI und wiederkehrendem Anleitungsmaterial reduziert dies doppelte Arbeit und verbessert im Lauf der Zeit die Konsistenz.

Translation Memory wird oft zusammen mit Glossaren besprochen, aber beides ist nicht identisch. Ein Glossar definiert bevorzugte Begriffe. Translation Memory hält fest, wie größere Segmente im Kontext übersetzt wurden. Zusammen helfen sie Übersetzern, schneller zu arbeiten und zugleich die Formulierung über Releases hinweg stabil zu halten.

Auch in KI-gestützten Workflows bleibt Translation Memory nützlich. Generative Werkzeuge können jedes Mal einen neuen Entwurf erzeugen, aber das garantiert keine stabile Terminologie oder Formulierung. Translation Memory gibt Teams eine wiederholbare Ausgangsbasis und hilft Reviewern zu erkennen, wann neuer Output ohne guten Grund von etabliertem Gebrauch abweicht.

Das ist am wichtigsten, wenn Inhalte kontinuierlich aktualisiert werden. Ein langfristiges mehrsprachiges Projekt braucht mehr als einmalige Übersetzungsqualität. Es braucht redaktionelles Gedächtnis. Deshalb bleibt der Wiki-Eintrag zu [Translation Memory und Glossaren](/docs/wiki/de/translation-memory-and-glossaries.html) auch in einem Repository relevant, das ebenfalls KI-gestützte Übersetzung behandelt.

Für Let Books ist der Wert praktisch. Der Dokumentationskorpus umfasst Produkt-, Barrierefreiheits-, Lokalisierungs- und Metadaten-Themen. Manche Formulierungen sollten über Sprachen und Zeit hinweg stabil bleiben. Translation Memory ist eines der Werkzeuge, das hilft, diese Stabilität sichtbar zu halten, statt sich auf informelle Erinnerung zu verlassen.
