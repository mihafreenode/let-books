# Memoria di traduzione

La translation memory è un sistema che memorizza segmenti tradotti in precedenza così da poterli riutilizzare quando ricompare lo stesso testo o un testo simile. Nella documentazione, nell'interfaccia di prodotto e nel materiale istruttivo ricorrente, questo riduce il lavoro duplicato e migliora la coerenza nel tempo.

La translation memory viene spesso discussa insieme ai glossari, ma le due cose non sono identiche. Un glossario definisce i termini preferiti. La translation memory registra come segmenti più grandi sono stati tradotti nel contesto. Insieme aiutano i traduttori a lavorare più velocemente mantenendo stabile la formulazione tra un rilascio e l'altro.

Anche nei workflow assistiti dall'IA, la translation memory resta utile. Gli strumenti generativi possono produrre ogni volta una bozza nuova, ma questo non garantisce terminologia o formulazioni stabili. La translation memory offre ai team una base ripetibile e aiuta i revisori a vedere quando un nuovo output diverge dall'uso consolidato senza una buona ragione.

Questo conta soprattutto quando i contenuti vengono aggiornati in modo continuo. Un progetto multilingue di lunga durata ha bisogno di più della qualità di una traduzione una tantum. Ha bisogno di memoria editoriale. Per questo la voce wiki su [Translation Memory e glossari](/docs/wiki/it/translation-memory-and-glossaries.html) resta rilevante anche in un repository che discute anche di traduzione assistita dall'IA.

Per Let Books il valore è pratico. Il corpus documentale abbraccia prodotto, accessibilità, localizzazione e temi di metadati. Alcune espressioni dovrebbero restare stabili tra lingue e nel tempo. La translation memory è uno degli strumenti che aiutano a mantenere visibile questa stabilità invece di affidarsi a ricordi informali.
