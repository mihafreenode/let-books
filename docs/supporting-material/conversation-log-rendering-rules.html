Conversation Log Rendering Rules

This document describes how the HTML conversation log should render different content types.

1. User Messages

Render as normal chat bubbles or conversation entries.

2. Assistant Discussion

Render Markdown as HTML.

3. Prompt Artifacts

Do not render Markdown. Preserve the raw text inside a collapsible section.

Prompt: Extract Portable Engineering Skills
# Extract Portable Engineering Skills
...

4. Pasted Agent Output

Do not render as Markdown. Preserve terminal/OpenCode styling:

  • dark background
  • monospace font
  • preserved indentation
  • preserved spacing
  • scrollable blocks
  • collapsible sections
Thinking: Streamlining Skills Assessment

1. Localization Governance
2. Documentation-as-Product
3. Validator Engineering
...

Recommended Rendering Matrix

TypeRender Style
User messageNormal chat bubble
Assistant discussionMarkdown → HTML
Prompt artifactCollapsible raw artifact block
Agent outputCollapsible OpenCode terminal block
Final conclusionsMarkdown → HTML
