# Sources

## Summary

`docs/sources/` will hold source maps and research notes for future articles and guides.

## Purpose

Every substantial claim should be traceable through repository evidence, external sources, and editorial notes.

## Repository Evidence Rules

- For blog posts and source maps, repository evidence must come from specifications and documentation.
- Prefer `docs/**`, `AGENTS.md`, `AGENTS-Implementation.md`, and other documentation files over source code paths.
- Do not use `static-demo/` or future application source files as quoted editorial evidence.
- If a feature is already implemented in a current build, record that as implementation status while keeping docs/specs as the canonical citation.

## Structure

- source maps by article or topic
- research notes by language when needed
- links to standards, specs, official docs, and academic references
- `source-map-template.md` for new article source maps
