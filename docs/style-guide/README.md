# Style Guide

## Summary

`docs/style-guide/` defines how LetBooks knowledge content should be written, localized, reviewed, and published.

## Purpose

This area keeps the knowledge platform coherent as it grows across languages and content types.

## Structure

- `terminology/` for term control
- `publishing/` for workflow and tooling rules
- `localization/` for translation rules
- `writing/` for tone and structure rules
