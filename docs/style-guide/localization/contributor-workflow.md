# Contributor Workflow

## Summary

This workflow is for translators, reviewers, librarians, educators, and other non-core-code contributors.

Before translating, updating, or reviewing localized content, contributors must review relevant native-speaker findings, terminology guidance, and localization review rules.

## Steps

1. choose a page or article
2. check the canonical English source
3. review terminology notes, relevant native-speaker findings, and applicable localization review rules
4. explicitly consider whether known defect classes apply to the content being changed
5. apply translation or review changes
6. check whether the change reveals reusable guidance, validator, or terminology updates
7. record maturity, review status, defect classes considered, and any findings that influenced decisions
