# Markdown во развој на производ

## Резиме

Markdown е практичен формат за чување на намерата на производот, документацијата, водичите и доказите во форма што е прегледна, пренослива, споредлива преку diff-ови и лесно поврзана со текови на валидација.

## Зошто е важен Markdown

Markdown често се третира како практичен формат за пишување. Во работа водена од спецификација тој е покорисен од тоа.

На тимовите им дава лесен начин знаењето за производот да остане блиску до репозиториумот без да биде скриено во сопственички алатки или тешко прегледни документи.

Тоа е важно за:

- спецификации
- имплементациски насоки
- документација за тековна состојба
- водичи и wiki страници
- мапи на извори и белешки за докази

## Во што е добар Markdown

Markdown добро функционира кога содржината треба да биде:

- читлива во сурова форма
- под контрола на верзии
- лесна за преглед во diff-ови
- поврзлива меѓу документи
- пренослива меѓу околини

## Што Markdown не прави сам по себе

Markdown е корисен, но сам по себе не обезбедува исправност.

Не може сам да гарантира:

- дека содржината е тековна
- дека тврдењата се добро поткрепени со извори
- дека имплементацијата се совпаѓа со документацијата
- дека правилата за валидација или CI навистина работат

## Контекст Let Books

Репозиториумот Let Books веќе користи Markdown за повеќе слоеви на меморија на производот, вклучувајќи:

- `AGENTS.md`
- `AGENTS-Implementation.md`
- `README.md`
- подрачни README датотеки и водичи
- мапи на извори во `docs/sources/`
