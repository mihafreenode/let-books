# Markdown у развоју производа

## Сажетак

Markdown је практичан формат за чување намере производа, документације, водича и доказа у облику који је прегледан, преносив, упоредив кроз diff-ове и лако повезан са валидационим токовима рада.

## Зашто је Markdown важан

Markdown се често третира као практичан формат за писање. У раду вођеном спецификацијом кориснији је од тога.

Тимовима даје лак начин да знање о производу остане близу репозиторијума без скривања у власничким алатима или тешко прегледним документима.

То је важно за:

- спецификације
- имплементациона упутства
- документацију тренутног стања
- водиче и wiki странице
- мапе извора и белешке о доказима

## У чему је Markdown добар

Markdown добро функционише када садржај треба да буде:

- читљив у сировом облику
- под контролом верзија
- лак за преглед у diff-овима
- повезив међу документима
- преносив између окружења

## Шта Markdown не ради сам од себе

Markdown је користан, али сам по себи не обезбеђује исправност.

Не може сам да гарантује:

- да је садржај актуелан
- да су тврдње добро поткрепљене изворима
- да се имплементација подудара са документацијом
- да правила валидације или CI-ја заиста раде

## Контекст Let Books

Репозиторијум Let Books већ користи Markdown за више слојева меморије производа, укључујући:

- `AGENTS.md`
- `AGENTS-Implementation.md`
- `README.md`
- подручне README датотеке и водиче
- мапе извора у `docs/sources/`
