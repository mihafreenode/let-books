# Wiki

## Summary

`docs/wiki/` will hold evergreen reference material.

## Purpose

Use this area for stable concept pages, glossary-style references, and short linked explainers that are updated over time.

## Structure

- topic pages
- glossary pages
- architecture reference pages
- workflow reference pages
- `index.html` per locale directory — generated by `tools/generate-index-pages.mjs`; lists all wiki pages for that locale

## Frontmatter

Every wiki page Markdown file has YAML frontmatter with:
- `title` — the page's H1 title
- `summary` — a one-sentence description (from the `## Summary` section)
- `topics` — an array of topic identifiers for cross-referencing with blog and learning content

Frontmatter is added by `tools/add-md-frontmatter.mjs`.

## Topic Navigation

- Topic identifiers are shared across blog, learning, and wiki content to enable future cross-linking.
- Blog article index pages group content by these topics.
