Kako održati usklađenost između specifikacije, dokumentacije, demoa i koda

Ovaj vodič opisuje praktičan postupak za održavanje usklađenosti između proizvodne specifikacije, dokumentacije, demoa i implementacije tijekom vremena.

Teme: Usklađenost demoa Dokumentacija Razvoj vođen specifikacijom

Preporučeni red autoriteta

  1. proizvodna specifikacija
  2. implementacijske smjernice
  3. dokumentacija trenutačnog stanja
  4. demo ili ponašanje aplikacije
  5. testovi i validacija

Preporučeni radni tok

  • najprije ažuriraj specifikaciju ili smjernice
  • zatim ažuriraj dokumentaciju trenutačnog stanja
  • tek nakon toga mijenjaj demo ili kod
  • dodaj provjeru ili test gdje je moguće

Što učiniti kada demo pokaže bolje rješenje

Ako demo pokaže bolji tok, to nije razlog da sam postane novi izvor istine. Najprije donesi odluku, zatim ažuriraj specifikaciju i dokumentaciju, a potom učvrsti promjenu u implementaciji.

Ključno pravilo

Ne provjeravaj samo radi li promjena. Provjeravaj i ostaje li usklađena s dokumentiranom namjerom proizvoda.
