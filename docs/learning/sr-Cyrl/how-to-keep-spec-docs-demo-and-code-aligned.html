Како одржати усклађеност између спецификације, документације, дема и кода

Овај водич описује практичан поступак за одржавање усклађености између спецификације, документације, дема и имплементације.

Теме: Усклађеност демоа Документација Развој вођен спецификацијама

Ред ауторитета

  1. производна спецификација
  2. имплементациона упутства
  3. документација тренутног стања
  4. демо или понашање апликације
  5. тестови и валидација

Практично правило

Најпре ажурирај спецификацију, затим документацију, па тек онда демо или код. Ако демо покаже бољи ток, одлука мора бити записана, а не само виђена.
