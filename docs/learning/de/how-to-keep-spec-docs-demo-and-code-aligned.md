# Wie Man Spezifikation, Dokumentation, Demo Und Code Im Einklang Hält

## Zusammenfassung

Dieser Leitfaden beschreibt einen praktischen Weg, Spezifikation, Dokumentation, Demo und Implementierung aufeinander abgestimmt zu halten.

## Autoritätsreihenfolge

1. Produktspezifikation
2. Implementierungsleitlinien
3. Dokumentation des aktuellen Zustands
4. Demo oder Anwendungsverhalten
5. Tests und Validierung

## Regel

Zuerst Spezifikation aktualisieren, dann Dokumentation, danach erst Demo oder Code. Wenn ein Demo einen besseren Ablauf zeigt, muss die Entscheidung dokumentiert werden.
