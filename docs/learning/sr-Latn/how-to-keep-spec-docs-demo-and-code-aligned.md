# Kako Održati Usklađenost Između Specifikacije, Dokumentacije, Dema I Koda

## Sažetak

Ovaj vodič opisuje praktičan postupak za održavanje usklađenosti između specifikacije, dokumentacije, dema i implementacije.

## Red autoriteta

1. proizvodna specifikacija
2. implementaciona uputstva
3. dokumentacija trenutnog stanja
4. demo ili ponašanje aplikacije
5. testovi i validacija

## Praktično pravilo

Najpre ažuriraj specifikaciju, zatim dokumentaciju, pa tek onda demo ili kod. Ako demo pokaže bolji tok, odluka mora biti zapisana, a ne samo viđena.
