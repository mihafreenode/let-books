# Како Да Се Одржи Усогласеност Меѓу Спецификација, Документација, Демо И Код

## Резиме

Овој водич опишува практичен процес за одржување усогласеност меѓу спецификацијата, документацијата, демото и имплементацијата.

## Ред на авторитет

1. продуктна спецификација
2. имплементациски насоки
3. документација за тековна состојба
4. демо или однесување на апликацијата
5. тестови и валидација

## Практично правило

Прво ажурирај спецификација, потоа документација, па дури потоа демо или код. Ако демото покаже подобар тек, одлуката мора да биде запишана.
