# Comment garder alignés spécification, documentation, démo et code

## Résumé

Ce guide décrit une manière pratique de garder alignés la spécification, la documentation, la démo et l'implémentation.

## Ordre d'autorité

1. spécification produit
2. directives d'implémentation
3. documentation de l'état actuel
4. démo ou comportement de l'application
5. tests et validation

## Règle

Mettez d'abord à jour la spécification, puis la documentation, puis seulement la démo ou le code. Si la démo montre un meilleur flux, la décision doit être enregistrée.
