Cómo mantener alineados la especificación, la documentación, la demo y el código

Esta guía describe una forma práctica de mantener alineados la especificación, la documentación, la demo y la implementación.

Temas: Alineación de la demo Documentación Desarrollo guiado por especificaciones

Orden de autoridad

  1. especificación del producto
  2. directrices de implementación
  3. documentación del estado actual
  4. demo o comportamiento de la aplicación
  5. pruebas y validación

Regla

Primero actualiza la especificación, luego la documentación y solo después la demo o el código. Si la demo muestra un flujo mejor, la decisión debe quedar registrada.
