# Cómo convertir una especificación de producto en un plan de implementación

## Resumen

Esta guía muestra cómo convertir una especificación de producto en un plan de implementación concreto sin perder propósito, límites y reglas esenciales.

## Puntos clave

- propósito del producto
- flujo de usuario
- límites del alcance
- reglas no negociables
- comportamiento de respaldo
- criterios de aceptación

## Tres niveles

1. nivel de flujo
2. nivel de sistema
3. nivel de entrega

## Preguntas de control

- se conserva el flujo de usuario
- se incluyen documentación y validación
- otra persona puede seguir el plan sin adivinar
