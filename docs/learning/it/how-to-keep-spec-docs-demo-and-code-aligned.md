# Come mantenere allineati specifica, documentazione, demo e codice

## Riepilogo

Questa guida descrive un modo pratico per mantenere allineati specifica, documentazione, demo e implementazione.

## Ordine di autorità

1. specifica di prodotto
2. linee guida di implementazione
3. documentazione dello stato attuale
4. demo o comportamento dell'applicazione
5. test e validazione

## Regola

Prima aggiorna la specifica, poi la documentazione, e solo dopo demo o codice. Se la demo mostra un flusso migliore, la decisione deve essere registrata.
