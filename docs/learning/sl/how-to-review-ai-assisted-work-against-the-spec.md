# Kako pregledovati AI-podprto delo glede na specifikacijo

## Povzetek

Ta vodič pojasni, kako AI-podprto implementacijo pregledovati proti specifikaciji, pravilom toka, dokumentaciji in validaciji, ne le po videzu ali tehnični verjetnosti.

## Vrstni red pregleda

1. produktna usklajenost
2. usklajenost dokumentacije
3. rezervno vedenje in validacija
4. tehnična kakovost

## Pogoste pasti

- poliran izhod se zdi pravilen
- najbližji obstoječi vzorec je že lahko napačen
- AI doda nenačrtovan obseg
- testi potrjujejo generirano vedenje, ne nujno nameravanega

## Praktična vprašanja

- katero pravilo specifikacije naj bi sprememba uresničila
- ali ohranja meje produkta
- ali so bili dokumenti posodobljeni
- ali ostaja rezervni tok uporaben
- kaj od dodanega je zunaj obsega

## Glavna lekcija

Najboljše vprašanje pri AI-pregledu ni "ali deluje", temveč "ali je še vedno pravi izdelek".
