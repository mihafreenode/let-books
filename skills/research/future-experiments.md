# Future Experiments

- Install `engineering-governance.md` into a small internal application repository with no multilingual content and measure the minimum useful footprint.
- Install `localization-governance.md` into a different multilingual repository and compare how much of the findings-corpus model transfers unchanged.
- Install `documentation-as-product.md` into a software repository whose docs are not statically generated and measure which validation pieces still matter.
- Install `ci-cd-governance.md` into a non-GitHub CI platform and compare the adaptation cost of rebuild-before-publish and workflow self-validation.
- Compare whether a docs-heavy repository wants `documentation-as-product.md` before `ci-cd-governance.md`, or whether the reverse order is more practical.
- Test whether a thin starter package for governance, without validators, still changes contributor behavior in a meaningful way.
- Evaluate whether supporting-material transcript logs should become an optional governance artifact in future repositories that want evidence trails for AI-assisted architecture and policy work.
