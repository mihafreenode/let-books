# Surprises

- `Validator Engineering` looked like a standalone skill at first, but repeated evidence showed it is mostly an enforcement and knowledge-preservation pattern inside broader governance.
- The localization system is more mature than the parent governance template because it has a stronger explicit feedback loop.
- Workflow documentation in `.github/workflows/README.md` is unusually rich and reads more like executable policy than a normal workflow catalog.
- The repository uses the same governance logic for docs, tests, tools, and workflows, which makes the meta-skill interpretation stronger than the flat peer model.
